                # Frequently Asked Questions (FAQ)

                # Connection

                # How to proceed when connection is lost between PC and device?

                A connection loss can happen due to various reasons like physical disconnect of the cable or device shutdown from the main power switch. In this case the connection status is not updated in real time in the status bar and a software restart is needed.

                # What happens if a sensor is disconnected during profile run?

                The application constantly monitors sensors values and if a value decreases abruptly, the currently running profile is automatically stopped by the application.

                # What happens to the component profiles when the underheater type is changed?

                If underheater type is changed, the old underheater profile is lost and a new default profile is generated for the current underheater type. These values have to be adjusted according to the current underheater type.

                # Calibration

                # How can calibration quality be improved?

                Calibration works best when the white dot on the nozzle (called "blob") is perfectly visible to the camera. In order to improve the vision of the camera

                • Remove PCBs from under the camera. This will reduce the possibility that solder contacts are falsely recognized as blob.
                • Turn laser off. This will reduce the possibility that the laser dot is falsely recognized as blob.
                • Make sure that there is enough light in the room.
                • For the Maxi BGA XL lens, it is highly recommended to take as many pictures of the chessboard as possible, in as many angles and positions as possible

                # How often is calibration needed?

                Calibration should be run

                • After application is installed or re-installed;
                • When a new PC or tablet is used;
                • Every time when a lens is used the first time with a device
                • Every time when the lens is replaced with another one
                • After a larger impact on the rework station arm, that could influence the angle between the camera and the rest of the arm
                • Every time when placement accuracy seems degraded
                • Once a month

                # What happens when calibration fails?

                For some calibration steps that failed a set of default data will be used based on used lens type. Please note that some functionalities will be affected, especially placement which might not work properly if these default values will be used.

                Make sure that the following are checked

                • No board is installed over the underheater before running calibration
                • The optimal light level is set correctly, usually 2
                • The white dot on the nozzle (called "blob") is visible to the camera
                • The proper placement nozzle is used for the current camera
                • In special cases, when using
                  • Maxi BGA XL lens, the following must be enabled from File menu Settings option
                    • Maxi BGA XL lens
                    • BGA/QFP 15 nozzle
                  • Flip-Chip lens, the light level should be set to the maximum possible 8. Otherwise, when performing the perspective calibration, the blob on the nozzle might be missed, thus resulting in inconsistent calibration data or failure
                • The device does not has any physical fault

                # Where is calibration data stored?

                The data obtained through calibration is stored locally, on the used PC or tablet. Therefore, changing the PC or tablet will result in loss of calibration data, thus requiring a new calibration.

                # Underheater auto-profiler

                # What could be the main reason for underheater auto-profile failure?

                Causes of underheater auto-profiling failure

                • The sensor might not be placed correctly or close to the area where the board is being heated
                • In case if the temperature of the board increases quite slowly, verify the sensor
                • Please also note that a failure can happen because of the safety check mechanism, that aborts the process after ~30 seconds if it detects there is no or a slight increase in temperature, but this is a normal behaviour
                • The problem may be hardware related, inspect the underheater control unit of the device

                # Error reporting

                # If a bug is found in the application, how to report it?

                We thank you for taking the time to report a bug and appreciate your input.

                Please send your bug description using our contact info. Make sure that you include the following things in the report

                1. A detailed description of the encountered bug
                2. Zipped up content of the Logs folder, that contains log files and calibration diagnostics
                  • The Logs folder can be opened from the Help menu, Open logs folder submenu
